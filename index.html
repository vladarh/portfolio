Начало профессионального пути

Во время обучения в университете один из преподавателей попросил протестировать их продукт. Я провёл атаку и получил полный admin доступ ко всем конфиденциальным данным системы.

Потратил целый день, исследуя архитектуру и уязвимости. Когда это было обнаружено, провели расследование и опрос. После некоторых колебаний, я признался в своих действиях.

Это стало поворотным моментом: мне предоставили возможность поговорить с CEO компании, завершить атаку с полной разведкой всех уязвимостей и написать профессиональный отчёт.

Вскоре поступило предложение на позицию Security Evaluator. Так началась моя профессиональная карьера в информационной безопасности.

Честность и профессионализм всегда окупаются. Этот случай научил меня, что ethical hacking — это не только про технические навыки, но и про ответственность.

Решённые проблемы и вызовы

Проблема: 300+ часов на оценку соответствия

Что было:

  • Команде из 6 человек требовалось 300+ часов на одну оценку
  • Ручной анализ документации - 80 часов
  • Ручные проверки серверов по чек-листам
  • Повторная оценка несколько итераций

Решение:

  • Разработал полнофункциональную платформу с Django + Vue.js + Ansible
  • ML конвейеры для автоматической генерации ПиМИ (-40 часов)
  • Ansible playbooks для автоматической оценки Linux/Windows
  • Автогенерация отчётов в Excel с экспортом конфигураций
Результат: Время оценки сокращено на 50%, пропускная способность команды увеличена в 2 раза

Проблема: Миграция Helm-чартов между контурами

Что было:

  • Ручная миграция чартов занимала часы
  • Несовместимость форматов между системами
  • Captain-chartMuseum не поддерживал API для списка репозиториев
  • Provenance не подходил для внутренних нужд

Решение:

  • Разработал GitLab CI/CD компонент для автоматической миграции
  • Модифицировал исходный код Captain-chartMuseum: добавил API endpoint для получения списка репозиториев
  • Заменил provenance на cosign для подписи образов
  • Multi-source/target поддержка: GitLab, Gitea, Nexus, Captain-chartMuseum
Результат: Автоматическая миграция чартов между контурами, гарантия целостности

Проблема: CI/CD только для существующей инфраструктуры

Что было:

  • Ansible playbooks работали только с уже развернутой инфраструктурой
  • Невозможность быстрого деплоя stage окружения
  • Ручная настройка каждого нового окружения

Решение:

  • Переписал Ansible playbooks для универсального деплоя с нуля
  • Добавил поддержку как prod, так и stage окружений
  • Настроил Selectel инфраструктуру: 2 сервера + кластер БД
Результат: Деплой любого окружения с нуля за минуты, а не часы

Проблема: Выбор контейнерной безопасности для ЕЦПК

Что было:

  • Нужно было выбрать между Luntry и NeuVector
  • Недостаточно данных для принятия решения
  • Требовались испытания по ПиМИ

Решение:

  • Провёл развёртывание обоих решений на тестовых кластерах
  • Сравнительный анализ: detection rate, false positives, performance impact
  • Провёл испытания по ПиМИ с составлением отчёта соответствия
Результат: Аргументированный выбор решения, повышение эффективности обнаружения уязвимостей

Проблема: Отсутствие системы бэкапа кода

Что было:

  • Единственная точка отказа - внешний Git сервис
  • Нет локального бэкапа исходного кода
  • Windows Server без нативной поддержки Git серверов

Решение:

  • Развернул Forgejo на Windows Server с помощью WSL
  • Настроил автоматическую синхронизацию репозиториев
  • Создал зеркало исходного кода для disaster recovery
Результат: Надёжная система бэкапа, устойчивость к отказу внешних сервисов

Проблема: Ручная обработка изображений продуктов

Что было:

  • Ручное редактирование каждого изображения в фотошопе
  • Непостоянное качество и размеры
  • Часы на обработку каталога продуктов

Решение:

  • Разработал Python модуль для автоматической обработки
  • Функционал: сжатие, удаление фона, центрирование, масштабирование
  • Интеграция в backend для автоматической обработки при загрузке
Результат: Единообразное качество, автоматическая обработка за секунды
